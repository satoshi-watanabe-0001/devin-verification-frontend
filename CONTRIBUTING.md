# コントリビューションガイド

このリポジトリへのコントリビューションをご検討いただき、ありがとうございます。

## プルリクエスト（PR）について

### 言語

**全てのPRは日本語で作成してください。**

- **PRタイトル**: 日本語で記述
- **PR本文**: 日本語で記述
- **コメント**: 日本語で記述

詳細は[Devin PR言語ポリシー](https://github.com/satoshi-watanabe-0001/ahamo-dummy-demo2-system-design-docs/blob/main/07-ai-development-guide/DEVIN-PR-LANGUAGE-POLICY.md)を参照してください。

### PRテンプレート

このリポジトリには日本語のPRテンプレートが設定されています。PR作成時に自動的に表示されるテンプレートに従って記入してください。

テンプレートには以下のセクションが含まれています：

- **概要**: PRの目的と変更内容
- **実装内容**: 具体的な実装項目
- **技術的な詳細**: 技術スタック、設計判断
- **テスト結果**: 実行したテスト
- **スクリーンショット**: UI変更がある場合
- **セッション情報**: DevinセッションURLと依頼者情報
- **チェックリスト**: 確認事項

### 自動化

Devin AI Engineerが作成したPRには、自動的に日本語のサマリコメントが投稿されます（GitHub Actions使用）。

## コード規約

### Next.js / TypeScript

- TypeScriptの厳格モードを使用
- ESLintルールに準拠
- コンポーネントは`'use client'`ディレクティブを適切に使用
- 型定義は`types/`ディレクトリに配置

### スタイリング

- Tailwind CSSを使用
- レスポンシブデザインを考慮（モバイルファースト）
- アクセシビリティ（aria-label等）を実装

### コミットメッセージ

コミットメッセージは以下のフォーマットに従ってください：

```
<type>(<scope>): 日本語の説明

詳細な説明（必要な場合）
```

**type:**
- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント更新
- `test`: テスト追加/修正
- `chore`: その他の変更

## 開発環境

### セットアップ

```bash
# 依存関係のインストール
pnpm install

# 開発サーバーの起動
pnpm run dev

# ビルド
pnpm run build

# Lint
pnpm run lint

# 型チェック（package.jsonにスクリプトがある場合）
pnpm run typecheck
```

### テスト

変更を加えた場合は、以下を実行してください：

1. `pnpm run lint` - Lintエラーがないことを確認
2. `pnpm run build` - ビルドが成功することを確認
3. 手動テスト - ブラウザで動作確認
